# Mission: Frontend Component

## Objective
Build a reusable, accessible, tested UI component that handles its props, edge cases, and keyboard navigation.

## In scope
- The component file and its styles (CSS modules, Tailwind, or styled-components)
- Unit and/or snapshot tests
- Accessibility: keyboard navigation and ARIA labels on interactive elements

## Out of scope
- Backend integration (open a separate mission once the component is stable)
- Storybook setup (optional follow-up once the component ships)

## Definition of done
- Component renders without errors given valid props
- Empty state and error state render gracefully
- Interactive elements have ARIA labels
- Keyboard navigation works (tab, enter, escape where applicable)
- No console errors or warnings from the framework
- The backlog task is marked `done` with `passes: true`
